# 训练页 UI 改进计划

审查范围：移动端训练模式（`MobileCurrentExerciseView` 及其子组件）。

## 当前结构

```
MobileTrainingModeHeader   ← 固定顶部：训练模式 · 时间 · 组数 · % + 退出
MobileExerciseProgressCard ← 动作信息 + 进度 pills + 组选择横滑条
  └ MobileCurrentSetCard   ← 上次提示 + 重量/次数 + 快选 + RIR + 确认 + 组操作
MobileWorkoutBottomBar     ← 固定底部：信息行 + [上一个][休息][下一组]
```

---

## 问题清单（按优先级）

### P0 — 双主操作冲突（最严重）

录入流里有**两个绿色主按钮**，语义重叠、互相竞争：

- 卡片内：「确认本组，去下一组」（整行绿色）
- 底部 bar：「下一组 / 下一动作」（绿色）

用户填完一组后，不清楚该点哪个推进流程。两者都"前进"，但一个在卡片中部、一个在屏幕底部固定栏。

**影响**：核心交互路径有歧义，新用户困惑。

### P0 — 休息入口重复

底部 bar 同时有两个休息相关入口：

- 中间按钮「休息 90s」（直接开始/查看休息）
- 右上「休息 / 结束」展开按钮（面板内也是休息计时器）

两者功能重叠，占用底部宝贵空间。

### P1 — 首屏录入区被挤压

确认按钮位于：header + 动作标题 + 进度 pills + 组选择横滑条 + 上次提示 + 重量/次数 + 次数快选 + RIR 之后。在小屏（375×667）上，用户需要滚动才能看到「确认本组」，违背"录入即所见"原则。

### P1 — 组操作面板混杂两类操作

「组操作」折叠面板里混了：

- **组结构**：+组 / -组 / 新增动作
- **批量填充**：复制本组到空组 / 按上次填入空组

两类目的不同（编辑结构 vs 快速录入），堆在一起降低可读性。

### P2 — 「✓ 已达标」反馈太弱

达标时仅显示一行小绿字，缺乏即时正反馈。次数在目标区间是核心激励点，值得更明显的视觉强调。

### P2 — RIR 的 `?` 提示在触屏失效

RIR 旁的问号用 `Tooltip`（hover 触发），移动端无 hover，用户无法了解 RIR 含义。

### P2 — 底部展开面板「开始休息」间距挤压

`flex items-baseline gap-2` 让「组间休息」标签、「90s」、「开始休息」按钮在窄屏贴边。

### P3 — 信息冗余

- header 显示 `filledSets/totalSets · %`，底部 bar 又显示 `%`，重复。
- 底部 bar 信息行和展开面板头部都显示「动作名 · 第N组 · 状态」。

---

## 改进方案

### 方案 A：明确主操作（解决 P0 双主操作）

**核心原则**：一个屏幕只有一个"前进"主操作。

- **卡片内确认按钮**保留为唯一主操作（绿色、整行），它已根据上下文显示正确文案（"确认本组，去下一组" / "确认动作，去下一个"）。
- **底部 bar 的「下一组/下一动作」降级**为导航辅助：改为 `secondary` 样式（非绿色），定位为"跳过/手动切换"，与确认推进区分。
- 底部 bar 主色保留给真正的会话级操作（如结束训练入口），或全部用 secondary 弱化，让卡片确认按钮成为视觉焦点。

### 方案 B：合并休息入口（解决 P0 休息重复）

- 底部 bar 中间按钮直接承担休息全部功能：未休息时「休息 90s」一键开始；休息中显示倒计时并可点击展开调整（±15/30s、结束）。
- 移除右上独立的「休息 / 结束」展开按钮，或将其改为纯「结束训练」入口（去掉休息内容，只保留结束 + 完成提示）。

### 方案 C：精简首屏（解决 P1 挤压）

- 进度 pills（本动作 X/Y、还剩 N 组）信息已在右上角 `X/Y` 和组选择条体现，可移除或合并为一行更紧凑的展示。
- 「次数快选」按钮组在有"上次/快速调整"时为次要，可考虑并入快速调整面板，减少首屏垂直占用。
- 目标：375×667 屏幕下，重量/次数输入 + 确认按钮在首屏可见。

### 方案 D：拆分组操作面板（解决 P1 混杂）

折叠面板内分两组，加分隔与小标题：

- **本组录入辅助**：复制本组到空组、按上次填入空组、下一未填组（高频，靠上）
- **结构编辑**：+组/-组、新增动作（低频，靠下）

### 方案 E：强化达标反馈（解决 P2）

次数进入目标区间时：次数输入框边框/背景转绿 + 「✓ 已达标」徽章，提供更明确的即时反馈。

### 方案 F：RIR 说明改为可点击（解决 P2）

`?` 改为 `onClick` 切换一行说明文字（或用已有的 Toast/Popover），适配触屏。

### 方案 G：间距与冗余清理（解决 P2/P3）

- 修复展开面板休息行间距（用 `justify-between` 替代 `items-baseline gap-2`）。
- 统一信息展示，去掉 header 与底部 bar 的重复 `%`。

---

## 建议实施顺序

1. **P0 优先**：方案 A（主操作）+ 方案 B（休息合并）— 解决核心交互歧义，改动集中在 `MobileWorkoutBottomBar` 和 `MobileCurrentSetCard`。
2. **P1**：方案 C（首屏精简）+ 方案 D（面板拆分）。
3. **P2/P3**：方案 E/F/G — 细节打磨。

每步改完在移动视口（375 宽）用 preview 验证布局，确认无溢出、无报错。
