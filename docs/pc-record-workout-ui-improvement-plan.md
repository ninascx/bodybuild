# PC 端录入与训练页 UI 改进方案

## 目标

把 PC 端从“移动端布局放大版”调整为稳定、高密度、适合重复录入的工作台。核心目标不是增加功能，而是降低录入摩擦：

- 记录页：打开后能快速完成每日数据、备注、补充围度查看。
- 训练页：长动作列表下仍能快速定位、录入组数据、查看进度、控制休息和完成训练。
- PC 断点优先覆盖 `lg` 以上视口，重点检查 1366、1440、1920 宽度。

## 不触碰范围

这些是已确认的用户指定行为，后续实现不得重新改动：

- 不改记录页“输入后不要自动折叠，也不要后移”的稳定行为。
- 不改训记同步里自重动作数值导入逻辑。
- 不改移动端训练模式主流程。
- 不改数据结构、持久化格式、训记 Open API 请求格式。
- 不把目标判断、趋势分析塞回记录页；记录页仍保持高频录入工具定位。

## 当前依据

- 记录页入口：`src/tabs/DailyRecordTab.tsx`
- 每日常用录入：`src/components/daily/DailyEssentialsForm.tsx`
- 记录页补充面板：`src/components/daily/DailyRecordPanels.tsx`
- 训练页入口：`src/tabs/WorkoutTab.tsx`
- 训练控制区：`src/components/workout/WorkoutControlPanel.tsx`
- 动作记录卡：`src/components/workout/ExerciseRecordCard.tsx`
- 组录入行：`src/components/workout/ExerciseSetEditor.tsx`
- 动作工具栏：`src/components/workout/WorkoutRecordToolbar.tsx`

## 设计原则

1. PC 端以扫描和重复录入为主，不做营销页式大块视觉。
2. 主操作固定、次级操作收束，避免多个按钮竞争同一个下一步。
3. 长页面要有稳定定位工具：sticky 侧栏、动作跳转、当前进度。
4. 低频编辑留在折叠区，高频录入直接可见。
5. 布局变化只能发生在日期切换、进入/退出训练模式、显式展开折叠时，不能由输入值隐式触发。

## 记录页方案

### 信息架构

PC 端使用双栏工作台：

```text
┌──────────────────────────────────────────────────────────────┐
│ 日期工具条：日期切换 / 今天 / 保存状态 / 更多                 │
├──────────────────────────────────────┬───────────────────────┤
│ 主录入区                              │ 右侧补充区             │
│ - 每日记录 6 项                        │ - 最近 6 周日历         │
│ - 复制昨天 / 填入目标                  │ - 围度 / 更多记录       │
│ - 备注                                │ - 低频导出/复制入口     │
└──────────────────────────────────────┴───────────────────────┘
```

### 具体改进

1. `DailyRecordTab` 在 `lg` 以上改成 `grid-cols-[minmax(0,1fr)_360px]` 或相近比例；移动端保持当前单列。
2. 日期区从独立大块卡片改成 PC 横向工具条，减少顶部高度；保存状态靠右显示。
3. `DailyEssentialsForm` 在 PC 下固定 3 列 x 2 行，字段顺序由当前稳定逻辑提供，不因输入内容变化而重排。
4. “补充记录”在 PC 下可放进右栏，不再挤在每日记录卡内部；如果保留折叠，默认状态必须和当前字段稳定行为解耦。
5. 备注区在主栏给固定高度，例如 `min-h-28`，位置紧跟常用录入，适合每天补充感受。
6. 右栏用 sticky 布局，顶部与日期工具条对齐，滚动长页面时仍能看日历和围度。
7. “复制当前日期记录”和“导出此日”继续作为低频动作，不进入主录入区抢焦点。

### 记录页验收

- 输入体重、热量、蛋白、步数、睡眠、疲劳后，字段不折叠、不后移。
- 1366 宽度下主录入和右侧补充同时可见，无横向滚动。
- 备注区不被补充详情挤到首屏之外过远。
- 保存状态清晰，但不占用主 CTA 位置。
- 移动端现有路径不退化。

## 训练页方案

### 信息架构

PC 端使用训练工作台：

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部训练工具条：日期 / 训练名 / 状态 / 开始或继续 / 同步训记 │
├──────────────────────────────────────┬───────────────────────┤
│ 动作记录区                            │ Sticky 会话侧栏        │
│ - 动作快速跳转                         │ - 用时 / 休息           │
│ - 动作卡列表                           │ - 完成组 / 剩余组       │
│ - 每组横向录入                          │ - 当前动作 / 下一未完成 │
│ - 更多训练操作                          │ - 确认完成训练          │
└──────────────────────────────────────┴───────────────────────┘
```

### 顶部控制区

1. `WorkoutControlPanel` 在 PC 下压缩为一条“训练工具条”，只保留日期、训练标题、状态、保存状态、开始/继续、同步训记。
2. 训练未开始时，模板选择仍可见，但以“选择计划 + 开始训练”为主；模板预览默认收束，避免首屏过长。
3. 已有训练时，“更换训练计划”保持折叠，不进入主路径。
4. `WorkoutStatusOverview` 的统计信息移动到右侧会话栏，避免顶部重复显示状态。

### 动作记录区

1. 动作快速跳转条在 PC 下保持 sticky 或紧贴动作列表顶部，支持长训练快速定位。
2. 动作卡头部展示：动作名、完成组数、目标、上次最佳、展开状态。编辑动作、移动、删除放入卡内折叠编辑区。
3. `ExerciseSetEditor` 在 PC 下改成表格式横向行：

```text
组 | 重量 kg | 次数 | RIR | 上次同组/最佳 | 快速动作
1  | [ 80 ]  | [ 8 ] | 2  | 77.5kg x 8   | 套用 / 复制上一组
```

4. 重量微调、重量快选、目标次数快选不要在每一组都纵向铺满；PC 下可收成行内小按钮或只在 focus/hover 时显示。
5. 组之间保持稳定高度，避免出现提示文字后把下面所有组顶开。
6. “按上次填空组”“复制已填到空组”“增删组”保持在动作卡底部工具区，和逐组录入分层。

### Sticky 会话侧栏

1. 新增 PC-only 会话侧栏，复用现有训练状态、计时、休息和完成逻辑，不新建业务状态。
2. 侧栏内容：
   - 训练名和当前状态。
   - 用时、休息倒计时、休息控制。
   - 完成组、剩余组、完成率。
   - 当前动作 / 下一未完成动作快捷跳转。
   - 主要动作：继续训练或确认完成。
3. 侧栏 sticky 在 `top` 与应用头部留出间距；底部不能遮挡动作列表。
4. 侧栏只在 PC 显示；移动端继续使用现有训练模式和底部栏。

### 训练页验收

- PC 长列表录入时，不需要回到顶部才能看进度、休息或完成训练。
- 组录入在 1366 宽度下横向对齐，重量、次数、RIR 不重叠。
- 低频操作不抢主路径：模板管理、动作编辑、导出、保存模板都在折叠或更多区域。
- 已有训练模式逻辑不被拆散，移动端训练页行为不改变。
- 同步训记按钮仍可见，但不成为训练页唯一视觉焦点。

## 实施阶段

### P0：PC 外壳和边界

- 新增 PC 专用布局容器，不改数据流。
- 记录页改双栏，训练页预留右侧 sticky 会话栏。
- 保留移动端 class 路径。

验收：PC 双栏出现，移动端截图与当前结构一致。

### P1：记录页录入效率

- 日期区压缩成 PC 工具条。
- 常用录入固定 3 x 2。
- 备注区提升到主栏稳定位置。
- 补充详情进入右栏。

验收：输入字段后不折叠、不后移；1366 宽度无横向滚动。

### P2：训练页工作台

- 顶部控制区压缩。
- 会话侧栏接入现有训练计时、休息、完成状态。
- 动作跳转条与侧栏共同支持长列表定位。

验收：长训练页面可在任意滚动位置控制休息和完成训练。

### P3：组录入密度

- `ExerciseSetEditor` 增加 PC 横向布局。
- 上次记录提示、套用、复制上一组改成稳定行内呈现。
- 快选按钮在 PC 下收束，减少纵向空间。

验收：每个动作卡高度明显降低，连续录入不需要大量滚动。

### P4：视觉与无障碍收尾

- 检查焦点环、键盘 Tab 顺序、按钮可访问名称。
- 检查明暗主题对比度。
- 统一 PC 端 gap、边框、标题字号。

验收：无重叠、无文字溢出，键盘可完成日期、字段、组录入和完成训练。

- `src/tabs/WorkoutTab.tsx`：只负责训练页两栏编排。

## 验证计划

1. 静态检查：`npm run typecheck`。
2. 现有 lint 已有历史问题时，记录非本次引入项；新增文件不得产生新的 lint 错误。
3. 视觉检查：
   - PC：1366 x 768、1440 x 900、1920 x 1080。
   - 移动回归：390 x 844，确认主流程未改变。
4. 行为检查：
   - 记录页逐项输入后字段位置稳定。
   - 训练页 8 个动作、每个 4 组时，动作定位、组录入、休息、完成训练都可在当前滚动位置完成。
   - 训记同步自重动作仍能导入重量数值。

## 风险与处理

- 右侧 sticky 栏可能与全局导航或底部安全区冲突：使用明确 `top` 和 `max-height`，内部滚动只限侧栏。
- 动作卡表格式录入可能压缩过度：保留 `xl` 更宽布局，`lg` 可降级为两列。
- 模板选择若过度收起会影响新训练创建：未开始状态下保留计划选择，已开始后再收束。
- 记录页右栏如果内容过长：日历优先，围度和更多记录可继续折叠，但不得影响主录入字段位置。
